## 题意简述

有两个链表，可能从**某一点开始相交**。不存在环状结构，要求找到相交的起始节点。

---
## 我的思路：哈希表法

**核心思想：** 用哈希集合记录其中一条链表的所有节点，再遍历另一条链表查找首次出现的重复节点。
**步骤：**
1. 遍历链表 A，将所有节点存入哈希集合。
2. 遍历链表 B，若发现某节点已在集合中，则该节点即为相交起点。
3. 若遍历完 B 未找到，返回 null。
**复杂度：**
- 时间：O(m + n)
- 空间：O(m) 或 O(n)

---
## 题解思路：双指针法

**核心思想：** 让两个指针分别从两链表头出发，走到末尾后切换到另一链表头，最终会在相交点相遇（或同时为 null）。
**步骤：**
1. 指针 `pA` 从链表 A 头出发，指针 `pB` 从链表 B 头出发。
2. 同时向前移动一步：
    - 若 `pA` 到达末尾（null），则重定向到链表 B 的头。
    - 若 `pB` 到达末尾（null），则重定向到链表 A 的头。
3. 当 `pA == pB` 时停止，返回该节点（可能是相交点或 null）。
**原理分析：**
- 假设链表 A 独有部分长度为 `a`，链表 B 独有部分长度为 `b`，公共部分长度为 `n`。
- 指针 `pA` 走过的路径：`a + n + b`
- 指针 `pB` 走过的路径：`b + n + a`
- 两者路径长度相等，因此会同时到达相交起点。
- 若不相交（`n = 0`），两指针会在第二轮末尾同时为 null。

**复杂度：**
- 时间：O(m + n)
- 空间：O(1) ✨
